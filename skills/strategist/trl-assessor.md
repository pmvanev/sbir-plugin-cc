# TRL Assessment Methodology

## TRL Scale Reference

| TRL | Description | Evidence Required | SBIR Phase Fit |
|-----|-------------|-------------------|----------------|
| 1 | Basic principles observed | Published research, theoretical analysis | Phase I start |
| 2 | Technology concept formulated | Concept papers, analytical studies | Phase I start |
| 3 | Proof of concept | Lab experiments, simulations with synthetic data | Phase I target |
| 4 | Lab validation | Component testing in lab environment | Phase I exit / Phase II start |
| 5 | Component validation in relevant environment | Integrated component testing, field-relevant conditions | Phase II mid |
| 6 | System/subsystem prototype in relevant environment | Prototype demo in simulated operational conditions | Phase II target |
| 7 | System prototype in operational environment | Prototype tested in actual operational environment | Phase II exit |
| 8 | System complete and qualified | Full system tested and qualified | Phase III |
| 9 | System proven in operational environment | Operational deployment with mission data | Phase III |

## Assessment Process

### Step 1: Determine Current TRL
Source evidence from: prior publications | existing prototypes | test data | patent filings | past SBIR awards | company profile capabilities.

For each claimed TRL level, require at least one concrete evidence artifact. "We have experience in this area" is insufficient -- cite the specific work product.

### Step 2: Determine Target TRL
Parse solicitation language for advancement expectations:
- "demonstrate feasibility" -> TRL 3-4
- "develop prototype" -> TRL 5-6
- "validate in relevant environment" -> TRL 5-6
- "demonstrate in operational environment" -> TRL 7
- "transition to production" -> TRL 8-9

Cross-reference with: evaluation criteria weighting | TPOC guidance on expectations | agency history for similar topics.

### Step 3: Gap Feasibility Analysis
Assess whether the TRL gap is achievable within the proposed phase:

| Gap | Phase I Feasibility | Phase II Feasibility |
|-----|--------------------|--------------------|
| 1 level | High | High |
| 2 levels | Moderate-High | High |
| 3 levels | Low-Moderate (needs strong justification) | Moderate-High |
| 4+ levels | Not credible | Moderate (needs detailed roadmap) |

Factors that increase credibility: parallel prior work reducing risk | existing test infrastructure | experienced team with relevant past performance | partner contributions closing specific gaps.

Factors that decrease credibility: no prior work at current TRL | novel materials or processes required | regulatory approvals needed | dependency on external deliverables.

### Step 4: Construct Justification Narrative
Structure:
1. Current TRL with evidence citation
2. Target TRL with solicitation language citation
3. Gap assessment with feasibility rationale
4. Milestone mapping: what specific work advances each TRL increment
5. Risk factors that could impede progression

## Red Flags

- Claiming TRL 2->6 in Phase I (too ambitious for 6-12 months and $250K)
- Claiming TRL 5->6 in Phase I (insufficient novelty to justify SBIR funding)
- No evidence artifacts for claimed current TRL
- Target TRL misaligned with solicitation evaluation criteria
- TRL advancement plan that skips levels without justification
- Claiming TRL advancement without identifying required test infrastructure

## TPOC Integration Points

When TPOC data is available, adjust assessment for:
- Agency expectations for TRL at phase exit (TPOC may indicate higher/lower than solicitation text suggests)
- Specific demonstration requirements mentioned in conversation
- Prior program TRL benchmarks the agency uses as reference points
- Acceptability of simulation vs. physical prototype at each level
