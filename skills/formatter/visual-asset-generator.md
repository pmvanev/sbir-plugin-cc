# Visual Asset Generator

## Figure Types and When to Use Each

| Type | When | Characteristics |
|------|------|-----------------|
| System diagram | Show overall system architecture, subsystem relationships | Boxes, arrows, labels. Clear hierarchy. |
| Block diagram | Show functional decomposition, data flow between components | Input/output blocks, signal flow paths |
| Timeline / Gantt | Show project phases, milestones, deliverable schedule | Horizontal bars, milestone markers, phase labels |
| Chart (bar/line/pie) | Present quantitative data -- performance metrics, market sizing, budget allocation | Axes labeled with units, data sourced from research or budget |
| Concept image | Illustrate abstract concepts, operational scenarios, deployment environments | Descriptive, may reference external generation tools |
| Process flow | Show step-by-step procedures, decision logic, test protocols | Flowchart nodes, decision diamonds, swim lanes |
| Comparison table (visual) | Side-by-side comparison of approaches, technologies, or competitors | Structured grid with visual indicators (checkmarks, ratings) |

## Generation Methods

### SVG (Inline Generation)
Best for: precise technical diagrams, system architectures, block diagrams.
Approach: Generate SVG markup directly. Use consistent styling -- stroke widths, font families, color palette.
Constraints: Keep SVG under 200 lines. Use viewBox for scaling. Avoid embedded raster images.

### Mermaid
Best for: flowcharts, sequence diagrams, Gantt charts, class diagrams.
Approach: Write Mermaid syntax. Render via `mmdc` CLI if available, otherwise provide raw Mermaid for user to render.
Detection: Check for `mmdc` availability before attempting render.

```bash
# Check Mermaid CLI availability
which mmdc 2>/dev/null && echo "mermaid-available" || echo "mermaid-unavailable"
```

### Graphviz (DOT)
Best for: complex directed graphs, dependency trees, network topology.
Approach: Write DOT syntax. Render via `dot` CLI if available.
Detection: Check for `dot` availability before attempting render.

```bash
# Check Graphviz availability
which dot 2>/dev/null && echo "graphviz-available" || echo "graphviz-unavailable"
```

### Chart Generation
Best for: quantitative data visualization -- bar charts, line charts, pie charts.
Approach: Use Python matplotlib/seaborn if available, otherwise generate SVG chart markup directly.
Data source: Research artifacts, budget scaffolds, performance metrics from corpus.

### Nano Banana (Google Gemini Image Generation)
Best for: concept figures, technical illustrations, operational scenario visualizations, polished infographics, any visual that is not a pure structural diagram.
Approach: Craft a detailed prompt and invoke via `scripts/nano-banana-generate.sh`. Returns a PNG image.

**When to select Nano Banana over SVG/Mermaid:**
- The figure needs realistic or semi-realistic rendering (deployment environments, hardware concepts)
- The figure communicates an abstract concept that boxes-and-arrows cannot convey
- The evaluator benefits from a polished, professional illustration
- The figure includes complex scenes with multiple elements interacting

**Prompt engineering for SBIR figures:**
- Lead with the figure type: "Technical diagram showing...", "Concept illustration of...", "Infographic comparing..."
- Specify style: "Clean, professional, suitable for a government proposal. White background. No decorative elements."
- Include all labeled elements explicitly: "Label the components: sensor array, processing unit, communications module"
- Specify aspect ratio matching the proposal layout (typically 16:9 for full-width, 4:3 for column-width)
- Avoid: photorealistic people, logos, trademarked imagery, text-heavy layouts (use SVG for text-heavy)

**Usage:**
```bash
# Generate a concept figure
scripts/nano-banana-generate.sh \
  --prompt "Technical diagram of a compact directed energy system for maritime UAS defense. Clean, professional style suitable for a government SBIR proposal. White background. Show: phased array antenna, beam steering unit, power supply, cooling system, and target tracking sensor. Label each component clearly." \
  --output ./artifacts/wave-5-visuals/directed-energy-concept.png \
  --aspect-ratio 16:9 \
  --size 2K
```

**Detection:**
```bash
# Check if Nano Banana is available (API key configured)
test -n "$GEMINI_API_KEY" && echo "nano-banana-available" || echo "nano-banana-unavailable"
```

**Fallback:** If `GEMINI_API_KEY` is not set, fall back to writing an external brief for the user to generate manually.

### External Tool Integration (Manual Fallback)
Best for: figures that cannot be generated by any available tool, or when all API/CLI tools are unavailable.
Approach: Describe the figure specification in detail. Write to a brief file for the user to generate externally (Napkin.ai, Canva, etc). Track as "pending-external" in figure log.

## Figure Specification Format

Each figure gets a specification before generation:

```markdown
## Figure {N}: {Title}

- **Type**: {system-diagram | block-diagram | timeline | chart | concept | process-flow | comparison}
- **Method**: {svg | mermaid | graphviz | chart | nano-banana | external}
- **Purpose**: {What this figure communicates to the evaluator}
- **Content**: {Specific elements to include}
- **Cross-references**: {Section(s) that cite this figure}
- **Compliance items**: {Matrix item IDs this figure supports}
- **Caption**: {Draft caption text}
```

## Agency Format Requirements

Common requirements across SBIR solicitations:

| Requirement | Typical Constraint | Check Method |
|------------|-------------------|--------------|
| Resolution | 300 DPI minimum for raster | Check image metadata or ensure vector format |
| Color mode | Grayscale for some agencies; color for others | Read solicitation FORMAT requirements |
| File size | Individual figure caps (often 2-5 MB) | Check file size after generation |
| Format | PDF, PNG, or EPS depending on agency | Match solicitation spec |
| Labeling | Figure number, title, source citation required | Verify caption and numbering in document |
| Accessibility | Alt text or text descriptions for 508 compliance | Generate alt text for each figure |

## Figure Numbering Convention

Figures numbered sequentially per volume:
- Technical Volume: Figure 1, Figure 2, ...
- Cost Volume: Figure C-1, Figure C-2, ... (if any)
- Management Volume: Figure M-1, Figure M-2, ... (if any)

Cross-references in text use "Figure N" (capitalized). Every figure cited in text must exist. Every figure must be cited at least once.

## Iteration Pattern

1. Generate draft figure from specification
2. Write to `./artifacts/wave-5-visuals/{figure-name}.{ext}`
3. Write figure log entry to `./artifacts/wave-5-visuals/figure-log.md`
4. Present to user for review
5. Accept feedback: approve | revise (with notes) | regenerate (different method) | defer to external
6. On revision: update figure, update log entry, re-present
7. On external deferral: write specification brief to `./artifacts/wave-5-visuals/external-briefs/{figure-name}-brief.md`

## Figure Log Format

```markdown
# Figure Log

| # | Title | Type | Method | Status | File | Compliance Items |
|---|-------|------|--------|--------|------|-----------------|
| 1 | System Architecture | system-diagram | svg | approved | system-arch.svg | R-3, R-7 |
| 2 | Project Timeline | timeline | mermaid | pending-review | timeline.mmd | R-12 |
| 3 | Deployment Scenario | concept | nano-banana | approved | deployment-scenario.png | R-10 |
| 4 | Market Size | chart | external | pending-external | -- | R-15 |
```

Statuses: `draft` | `pending-review` | `revision-N` | `approved` | `pending-external` | `external-received`

## Method Selection Guide

| Figure Need | First Choice | Fallback |
|------------|-------------|----------|
| System architecture, block diagrams | Mermaid or SVG | SVG inline |
| Flowcharts, sequence diagrams | Mermaid | SVG inline |
| Dependency graphs, network topology | Graphviz | SVG inline |
| Data charts (bar, line, pie) | Python matplotlib | SVG inline |
| Concept figures, technical illustrations | Nano Banana | External brief |
| Deployment scenarios, operational environments | Nano Banana | External brief |
| Hardware/system visualizations | Nano Banana | External brief |
| Complex infographics | Nano Banana | External brief |
| Comparison tables (visual) | SVG | Mermaid |

## Color Palette for Consistent Styling

Use a restrained, professional palette across all generated figures:

- Primary: #2563EB (blue) -- main elements, headings
- Secondary: #059669 (green) -- positive indicators, completed items
- Accent: #D97706 (amber) -- warnings, attention items
- Neutral: #4B5563 (gray) -- borders, secondary text
- Background: #FFFFFF (white) -- clean background for print
- Text: #1F2937 (dark gray) -- readable on white background

Override palette only if agency solicitation specifies branding requirements.
