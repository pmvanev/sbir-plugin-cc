# JTBD Four Forces — sbir-developer-feedback

## Job 1 — Capture Friction In-Context

| Force | Detail |
|-------|--------|
| **Push** (current frustration) | When something goes wrong (bad image, wrong past performance, unexpected error), the user has to context-switch to write an email or Slack message, losing the moment and having to reconstruct state from memory. |
| **Pull** (desired future) | Run `/sbir:developer-feedback` right then — feedback is captured with a full state snapshot automatically. Nothing to reconstruct. |
| **Anxiety** (adoption concerns) | "Will this interrupt my workflow?" "Is it too complicated to fill out?" "Does it actually help or just disappear into a file nobody reads?" |
| **Habit** (behavior to change) | Silently working around issues, or making a mental note that gets forgotten, or posting a vague "the images looked bad" Slack message with no context. |

---

## Job 2 — Receive Actionable Feedback as Developer

| Force | Detail |
|-------|--------|
| **Push** | Getting vague feedback ("wave 3 felt broken") with no state context — can't reproduce, can't fix. |
| **Pull** | Feedback entries with: proposal topic, current wave, skipped waves, rigor profile, company profile age, finder results age, generated artifacts list, and ratings — everything needed to understand the exact session. |
| **Anxiety** | "Is the snapshot too noisy — will I have to sift through irrelevant data?" "Will sensitive company data (profile contents) end up in the feedback file?" |
| **Habit** | Either ignoring vague reports or scheduling a follow-up call to reconstruct the scenario. |

---

## Job 3 — Rate Specific AI Output Quality

| Force | Detail |
|-------|--------|
| **Push** | No current mechanism to signal "the images were terrible but everything else was fine" — feedback must be all-or-nothing prose. |
| **Pull** | Quick 1–5 rating per quality dimension (image quality, past performance relevance, writing quality, topic scoring accuracy) plus optional free text. Ratings aggregate over time to reveal patterns. |
| **Anxiety** | "Is 3/5 clear enough? What does it mean?" "Will my rating be misinterpreted?" |
| **Habit** | Skipping quality documentation entirely because there's no low-friction channel. |
